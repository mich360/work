About

１　　cd desktop

 2        django-admin startproject d_website (プロジェクト名)

          

 3       cd d_website


 4      MacBook-Pro d_website %　 python3 manage.py runserver

コマンドすると基本的な、adminのウェブができました。

同じ名前の特殊なフォルダと、manage.pyファイル（管理用）が自動的に作られます。

注意すべきは同じ名前の特殊なフォルダの中に、全体を設定するsettings.py と 

URLの設定ファイルurls.pyがある。まずはこの二つに作業加えるので重要です。


次に自分で表示するWebページを作る


このプロジェクトにwebsiteという名のアプリを作ります。

　　　　　python3 manage.py startapp website


全体設定ファイルつまりsettings.py を開いて”website”をセットする


”website”の中にtemplatesフォルダを新規に作りindex.htmlファイルを作る。

“website”の中 views.py  にhtmlを返しViewを追加する。

“website”の中 なければ作ったurls.py に　作ったViewとURLを結びつける。

全体設定ファイルのurls.pyにも観に行くよう(include)に書き加える。


GitHubにアップロードのやり方 

git init
git add .
git commit -m "commit"
git remote add origin リポジトリのURL
git fetch origin
git merge origin/main
git push -u origin main

fetch
   リモートリポジトリの最新の変更をローカルに取得し、変更内容や衝突の有無を確認することができます。
   fetchを省略した場合、リモートの変更を事前に把握できずに直接pushすることになります。
指定したリモートリポジトリが見つからないremote: Repository not found.出た時は
　　　　「　git remote set-url origin 新しいリポジトリのURL　」で改善する。
